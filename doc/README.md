# docs

## 文件分類（新結構）

- `docs/overview.md`：專案總覽、產品與本機啟動入口
- `docs/status`：目前狀態與 production blocker
- `docs/roadmap`：TODO、release checklist、任務索引與規劃筆記
- `docs/architecture`：架構、資料結構、持久化、Kafka 與 Redis 相關設計
- `docs/operations`：觀測、告警、SMTP、監控導向文件
- `docs/validation`：交易所功能驗收、demo 驗證與回歸檢查矩陣
- `docs/reliability`：風險控制、運維 runbook、可用性與操作流程
- `docs/runbooks`：災備、發布節點、跨系統重放與停機演練
- `docs/integrations`：外部整合、Polymarket
- `docs/ai`：Codex / Agent 協作文件（保留原結構）
- `docs/tasks`：任務中心（含 active 與 backlog；若目錄存在時使用）

## 建議閱讀順序

1. 先讀 `docs/overview.md`
2. 再讀 `docs/status/current-state.md`、`docs/roadmap/todo.md`
3. 需要實作則對照 `docs/roadmap/task-index.md` 或現有 `docs/tasks/README.md`
4. 查資料時以 `docs/DOCUMENTATION-INDEX.md` 的主題索引為主
