# P13-T02 - 建立 common error response / exception handling boundary

## 狀態

```text
completed
```

## 目標

建立後端共用錯誤回應格式與 exception boundary，讓後續 API 層可以用一致且不洩漏敏感資訊的方式回傳錯誤。

## Scope

```text
- 建立 common / api / error 類 package 的錯誤回應 DTO 與錯誤碼 enum
- 建立 global exception handler skeleton
- 補測試，驗證錯誤回應格式與分類
- 更新 boundary 文件
```

## Implementation rules

- 不要實作 runtime money movement。
- 不要實作 ledger posting。
- 不要實作 reservation hold / release。
- 不要實作 order matching。
- 不要實作 settlement。
- 不要實作 withdrawal signing / broadcast。
- 不要實作 deposit crediting。
- 不要新增或修改 Flyway migration。
- 不要改前端。
- 所有新增 public class / public method 必須有繁體中文維護性註解。
- 錯誤 response 不可洩漏 stack trace、SQL、secret、private key 或 wallet signing 資訊。
