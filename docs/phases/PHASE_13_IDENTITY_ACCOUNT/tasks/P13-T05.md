# P13-T05 - 建立 transaction boundary policy

## 狀態

```text
completed
```

## 目標

建立 transaction boundary 規範與 skeleton，讓後續 application service 可以依照 read-only / write transaction 與高風險流程標記進行設計。

## Scope

```text
- 新增 transaction boundary 文件
- 新增 transaction marker annotation / policy class / package skeleton
- 新增測試，驗證高風險 transaction boundary 不被誤用
- 更新 backend-module-boundary / persistence-boundary 文件
```

## Implementation rules

- 不要實作 runtime money movement。
- 不要實作 ledger posting。
- 不要實作 reservation hold / release。
- 不要實作 order matching。
- 不要實作 settlement。
- 不要實作 withdrawal signing / broadcast。
- 不要實作 deposit crediting。
- 不要新增或修改 Flyway migration。
- 不要改前端。
- 所有新增 public class / public method 必須有繁體中文維護性註解。
- transaction policy 不代表已實作 money movement。

```text
Run the focused transaction boundary test and the relevant Maven test compile.
```

## Risk

```text
Low-medium。主要風險是後續 application service 若忽略 policy，仍可能把高風險操作放錯 transaction boundary。
```
