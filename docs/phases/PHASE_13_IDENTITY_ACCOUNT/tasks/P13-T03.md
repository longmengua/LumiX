# P13-T03 - 建立 request validation 與 DTO convention

## 狀態

```text
completed
```

## 目標

建立後端 API request / response DTO 與 validation 的共用規範，讓後續 user、account、asset、market、wallet、order 等 API 先有一致命名與安全驗證邊界。

## Scope

```text
- 新增 request / response DTO 命名規則文件
- 新增 validation convention 文件
- 新增共用 validation helper / marker annotation / validation error detail DTO
- 補測試，驗證 validation error detail 不洩漏敏感資訊
- 更新 error-response-boundary 文件，說明 validation error 如何轉成 ApiErrorResponse
```

## Implementation rules

- 不要實作 runtime money movement。
- 不要實作 ledger posting。
- 不要實作 reservation hold / release。
- 不要實作 order matching。
- 不要實作 settlement。
- 不要實作 withdrawal signing / broadcast。
- 不要實作 deposit crediting。
- 不要新增或修改 Flyway migration。
- 不要改前端。
- 所有新增 public class / public method 必須有繁體中文維護性註解。

## Verification

```text
Run the focused validation boundary test and the relevant Maven test compile.
```

## Risk

```text
Low-medium。主要風險是未來 controller 實作若不遵守 convention，仍可能產生不一致 DTO。
```
