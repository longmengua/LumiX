# P13-T01 - 建立後端 package architecture 與 module boundary 文件 / skeleton

## 狀態

```text
completed
```

## 目標

把 LumiX 後端的 package 邊界先定義清楚，讓後續各 bounded context 可以依照固定 layer 演進，不再維持扁平且混雜的 package 入口。

## Required context

```text
AGENTS.md
AI_AGENT.md
AI_PROGRESS.md
docs/ai/AI_CONTEXT_ROUTING.md
docs/phases/PHASE_13_IDENTITY_ACCOUNT/README.md
server/docs/backend-module-boundary.md
```

## Scope

```text
- 檢查現有 server/src/main/java/com/lumix 目錄
- 建立或整理 backend module boundary 文件
- 新增 package skeleton 或 package marker
- 可新增 package-info.java 或 README 類文件
- 可新增只驗證 skeleton 的架構測試
```

## Implementation rules

- 不要實作 runtime money movement。
- 不要實作 ledger posting service。
- 不要實作 reservation hold / release service。
- 不要實作 order matching。
- 不要實作 settlement。
- 不要實作 withdrawal signing / broadcast。
- 不要實作 deposit crediting。
- 不要新增或修改 Flyway migration。
- 不要改前端。
- 新增或修改的程式碼必須使用繁體中文維護性註解。
- 架構圖只能用純文字圖。

## Verification

```text
Run the focused package boundary test and Maven test compile for the touched server sources.
```

## Risk

```text
Low。主要風險是 boundary 文件若與實際 package 命名不一致，後續 Phase 13 會需要再校正。
```
