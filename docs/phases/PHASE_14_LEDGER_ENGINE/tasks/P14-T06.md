## Scope

```text
- 新增 ledger persistence adapter skeleton / minimal implementation
- 新增 adapter-level integration test 或 repository test
- 新增文件，說明這只是 persistence adapter，不是 posting service
- 若新增真正 DB write，僅允許 append ledger_journals / ledger_entries，且只能由測試直接驗證
```

## Implementation rules

```text
- 不要新增 LedgerPostingService
- 不要把 LedgerPostingCommandBoundary 接到 DB adapter
- 不要新增完整 posting use case
- 不要新增完整 transaction orchestration
- 不要 insert / update / delete balance_projections
- 不要實作 reversal
- 不要實作 settlement
- 不要實作 reservation hold / release
- 不要實作 order matching
- 不要實作 withdrawal signing / broadcast
- 不要實作 idempotency runtime
- 不要實作 outbox publisher
- 不要實作 audit writer
- 不要新增或修改 Flyway migration
- 不要改前端
- 不要宣稱 production-ready
- 所有新增 public class / public method 必須有繁體中文維護性註解
- 所有 DB write 相關內容必須標示 HUMAN_REVIEW_REQUIRED
```

## Verification

```text
若新增 adapter implementation，必須有測試驗證只 append ledger_journals / ledger_entries。
測試必須驗證不能出現 update / delete ledger table。
測試必須驗證沒有 balance_projections mutation。
測試必須驗證沒有 LedgerPostingService。
測試必須驗證 posting command boundary 尚未接 DB adapter。
```

## Risk

```text
High。這是 Phase 14 第一個可能碰到 ledger DB append 實作的任務，任何接線都可能把 skeleton 誤升級成正式 posting runtime。
```
