completed

## 目標

只建立 ledger idempotency / request identity 的設計門檻，明確區分 requestId、idempotency key 與 business reference 的責任，不實作正式 runtime。

## Scope

```text
- 設計 request identity 與 idempotency 分工
- 建立 idempotency decision contract
- 建立 ledger idempotency design contract
- 驗證 requestId 只是 trace / correlation
- 驗證 business reference 不能取代 idempotency key
- 驗證 ledger posting boundary 尚未接 DB adapter
```

## Implementation rules

```text
- 不要實作正式 idempotency runtime
- 不要查詢或寫入 idempotency_keys
- 不要接 LedgerPostingCommandBoundary 到 DB adapter
- 不要新增 LedgerPostingService
- 不要新增 @Repository
- 不要新增 @Transactional
- 不要呼叫 JDBC / JPA / database client
- 不要修改 migration
- 不要更新 balance_projections
- 不要接 outbox / audit runtime
- 不要改前端
- 不要宣稱 production-ready
```

## Verification

```text
requestId 只能是 trace / correlation / audit linkage
idempotency key 才能表示 duplicate prevention contract
business reference 不能單獨取代 idempotency key
decision contract 必須能表達 NEW_REQUEST、DUPLICATE_COMPLETED、IN_PROGRESS、CONFLICTING_REQUEST、EXPIRED_OR_RETRY_REQUIRED
source-scan 不得出現 IdempotencyService、idempotency_keys SQL、outbox_events SQL、audit_logs SQL、@Repository、@Transactional、JdbcTemplate、EntityManager
source-scan 必須確認 LedgerPostingCommandBoundary 尚未接 LedgerAppendOnlyJdbcAdapter
P14-T02 到 P14-T07 既有測試仍通過
```

## Risk

```text
High。idempotency 一旦被誤解成完成的 runtime，後續會直接影響資金重送與重複執行風險。
```
