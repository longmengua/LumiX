## 目標

建立 runtime integration no-go gate，明確禁止把 ledger posting boundary 接到正式 runtime。

## Scope

```text
- 驗證 LedgerPostingCommandBoundary 沒有接 LedgerAppendOnlyJdbcAdapter
- 驗證 append adapter 只是 persistence gate，不是正式 posting runtime
- 驗證 idempotency / outbox / audit 仍只是設計
- 驗證 balance_projections 沒有被 ledger append adapter 更新
- 驗證沒有 LedgerPostingService
- 驗證沒有 @Repository / @Transactional
```

## Implementation rules

```text
- 不要新增正式 runtime
- 不要新增 IdempotencyService runtime
- 不要新增 OutboxPublisher
- 不要新增 AuditWriter
- 不要新增 LedgerPostingService
- 不要新增 @Repository
- 不要新增 @Transactional
- 不要呼叫 JDBC / JPA / database client
- 不要修改 migration
- 不要更新 balance_projections
- 不要改前端
```

## Verification

```text
source-scan 必須確認沒有 LedgerPostingService
source-scan 必須確認沒有 IdempotencyService runtime
source-scan 必須確認沒有 OutboxPublisher
source-scan 必須確認沒有 AuditWriter
source-scan 必須確認沒有 @Repository
source-scan 必須確認沒有 @Transactional
source-scan 必須確認 LedgerPostingCommandBoundary 沒有引用 LedgerAppendOnlyJdbcAdapter
source-scan 必須確認沒有 idempotency_keys / outbox_events / audit_logs runtime SQL
source-scan 必須確認沒有 balance_projections update / insert / delete
```

## Risk

```text
High。這個 gate 若被打破，ledger 將從設計 skeleton 直接跳到 runtime 接線，會放大資金與審計風險。
```
