## Scope

```text
- 建立 ledger posting integration design
- 定義 integration 順序
- 定義 accepted plan 與 posted / committed / persisted 的分工
- 確認 LedgerAppendOnlyJdbcAdapter 不可直接從 API / controller 呼叫
- source-scan 驗證沒有正式 posting runtime
```

## Implementation rules

```text
- 不要新增正式 ledger posting runtime
- 不要把 LedgerPostingCommandBoundary 接到 LedgerAppendOnlyJdbcAdapter
- 不要新增 LedgerPostingService
- 不要新增 @Repository
- 不要新增 @Transactional
- 不要新增 DB client call
- 不要修改 migration
- 不要更新 balance_projections
- 不要改前端
```

## Verification

```text
requestId 不等於完整 idempotency guarantee
accepted posting plan 不等於 posted / committed / persisted
integration order 必須是 idempotency -> prerequisite -> invariant -> append -> outbox -> audit -> reconciliation
source-scan 確認沒有 LedgerPostingService、@Repository、@Transactional
source-scan 確認 LedgerPostingCommandBoundary 沒有引用 LedgerAppendOnlyJdbcAdapter
source-scan 確認沒有 balance_projections / reservation / settlement runtime SQL
```

## Risk

```text
High。ledger posting integration 一旦誤接成 runtime，後續 money movement 風險會直接放大。
```
