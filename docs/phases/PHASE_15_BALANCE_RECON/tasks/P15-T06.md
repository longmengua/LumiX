## 目標

建立 reservation hold/release 的設計門檻，明確定義 reservation 對 available_amount / locked_amount 的未來語意，但不實作正式 reservation runtime。

## Scope

```text
- 新增 reservation hold/release 設計文件
- 新增 reservation design policy / enum / marker 型別
- 明確定義 hold / release / commit / cancel 的設計邊界
- source-scan 驗證沒有正式 reservation runtime
```

## Implementation rules

```text
- 不要新增 ReservationRuntimeService
- 不要新增 ReservationService
- 不要新增 BalanceMutationService
- 不要新增 BalanceProjectionService runtime
- 不要新增 SettlementService
- 不要新增 @Repository
- 不要新增 @Transactional
- 不要使用 JdbcTemplate
- 不要使用 EntityManager
- 不要 insert / update / delete reservations
- 不要 insert / update / delete balance_projections
- 不要 insert / update / delete ledger_journals
- 不要 insert / update / delete ledger_entries
- 不要接 order placement
- 不要接 matching
- 不要接 settlement
- 不要接 wallet withdrawal
- 不要修改 migration
- 不要改前端
```

## Verification

```text
hold 會降低 available_amount、增加 locked_amount
release 會增加 available_amount、降低 locked_amount
commit 必須走 settlement / ledger posting gate，不能直接等於 ledger transfer
requestId 不是 idempotency guarantee
matching / order 不得直接寫 reservation 或 balance_projections
source-scan 確認 main source 沒有 ReservationRuntimeService、ReservationService、BalanceMutationService、BalanceProjectionService、SettlementService、@Repository、@Transactional、JdbcTemplate、EntityManager
source-scan 確認沒有 INSERT / UPDATE / DELETE reservations、balance_projections、ledger_journals、ledger_entries runtime SQL
```

## Risk

```text
High。reservation 會直接影響 available / locked balance 語意，若設計門檻不清楚，後續交易 runtime 很容易把 hold / release 誤接成正式資金變動。
```
