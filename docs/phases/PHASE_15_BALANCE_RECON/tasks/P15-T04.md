## Scope

```text
- 新增受控 runtime gate
- 讓 accepted plan 進入 append adapter
- 將 rejected command 停在 DB append 之前
- 驗證 journal / entry append 與 rollback
- source-scan 驗證沒有正式 money movement runtime
```

## Implementation rules

```text
- 不要新增 LedgerPostingService
- 不要新增 BalanceMutationService
- 不要新增 BalanceProjectionService runtime
- 不要新增 ReservationRuntimeService
- 不要新增 SettlementService
- 不要新增 @Repository
- 不要新增 @Transactional
- 不要新增 DB client call 以外的正式 runtime
- 不要更新 balance_projections
- 不要接 idempotency_keys / outbox_events / audit_logs runtime
- 不要改前端
- 不要修改 migration
```

## Verification

```text
valid command 經受控 runtime 接線後，只 append ledger_journals / ledger_entries
rejected command 不得 append DB
entry insert 失敗時 journal header 必須 rollback
balance_projections row count 不變
不出現 idempotency_keys / outbox_events / audit_logs runtime SQL
LedgerPostingCommandBoundary 不得直接依賴 LedgerAppendOnlyJdbcAdapter
source-scan 確認沒有 LedgerPostingService、BalanceMutationService、BalanceProjectionService、ReservationRuntimeService、SettlementService、@Repository、@Transactional、JdbcTemplate、EntityManager
```

## Risk

```text
High。這是 Phase 15 第一個真正把 posting plan 接到 append adapter 的門檻，任何越界都會放大資金風險。
```
