## 目標

建立 balance projection rebuild runtime gate，讓 ledger_entries 可以被保守地重建成 balance_projections read model，但不把 balance_projections 當成資金 source of truth。

## Scope

```text
- 新增 balance projection rebuild gate
- 以 ledger_entries 為來源重建 balance_projections
- 明確採用 CREDIT 增加、DEBIT 減少的計算語意
- 目前只 materialize SPOT account projection rows
- available_amount 暫時等於 total_amount，locked_amount 暫時為 0
- projection_version 與 reconciled_at 必須寫入
- source-scan 驗證沒有 repository / transaction / JPA / JDBC template runtime
```

## Implementation rules

```text
- 不要新增 BalanceMutationService
- 不要把 balance_projections 當成資金真相來源
- 不要讓 order / matching / settlement 直接寫 balance_projections
- 不要新增 @Repository
- 不要新增 @Transactional
- 不要使用 JdbcTemplate
- 不要使用 EntityManager
- 不要更新 ledger_journals / ledger_entries
- 不要接 reservation hold / release
- 不要接 settlement
- 不要接 idempotency_keys / outbox_events / audit_logs runtime
- 不要改前端
- 不要修改 migration
```

## Verification

```text
valid ledger entries rebuild 成 balance_projections
多個 journal / entries 可彙總同一 account + asset
不同 asset / account 不可混算
CREDIT 增加 total_amount，DEBIT 減少 total_amount
rebuild 後 balance_projections 是 read model，不得反向修改 ledger_entries
ledger_journals / ledger_entries row count 不變
source-scan 確認沒有 BalanceMutationService、@Repository、@Transactional、JdbcTemplate、EntityManager
source-scan 確認沒有 order / matching / settlement 寫 balance_projections
```

## Risk

```text
High。這是 Phase 15 第一個把 ledger_entries 寫回 balance_projections 的 gate。
如果 rebuild 語意弄錯，read model 會直接偏離 ledger source of truth。
```
