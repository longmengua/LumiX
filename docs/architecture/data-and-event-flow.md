# 資料與事件流

LumiX 的正式環境資料不會只存在單一表或單一服務裡。它會經過下單、撮合、結算、帳本、餘額投影、行情與對帳等多個邊界。

這份文件直接描述架構層級的資料與事件流。目標是把生產方、消費方、命令、事件、持久化邊界、狀態變化與對帳點講清楚，讓這份文件單獨閱讀就能成立。

## 使用原則

- `C++ 撮合核心` 是撮合結果的權威來源。
- `帳本分錄` 是資金帳務的權威來源。
- `餘額投影` 是查詢用衍生視圖，不是資金真相本身。
- `市場資料` 只能從正式撮合事件長出來，不能從畫面或手工資料拼出來。
- `對帳` 必須能對回訂單、預留、成交、分錄、投影與市場資料序號。
- 重試、重複事件與重放都必須以冪等方式處理。

## 系統資料流

```text
[網頁客戶端 / Open API 客戶端]
    |
    | HTTP 請求
    v
[Spring Boot API]
    |
    | 驗證請求
    | 驗證身分
    | 驗證交易對
    v
[訂單服務]
    |
    | 建立訂單意圖
    | 載入帳戶狀態
    v
[預留引擎]
    |
    | 決定鎖定資產
    | 計算預留數量
    v
[訂單儲存層]
    |
    | 持久化訂單
    | 持久化預留
    v
[撮合閘道]
    |
    | 傳送命令到 C++ 撮合核心
    v
[C++ 撮合核心]
    |
    | 發出權威成交 / 拒絕 / 撤單事件
    v
[結算引擎]
    |
    | 計算資產變動
    | 計算手續費
    v
[帳本分錄]
    |
    | 持久化平衡的借貸分錄
    v
[餘額投影]
    |
    | 更新可用餘額
    | 更新鎖定餘額
    | 更新總餘額
    v
[資產查詢 API / 對帳]
```

## 系統事件流

```text
[客戶端請求]
    |
    | 訂單命令
    v
[訂單已接受事件]
    |
    | 訂單已持久化
    | 預留已持久化
    v
[撮合提交命令]
    |
    | 出站命令紀錄
    v
[撮合事件]
    |
    | 成交 / 拒絕 / 撤單
    v
[結算事件]
    |
    | 結算開始
    | 結算完成
    v
[帳本分錄事件]
    |
    | 分錄已提交
    v
[餘額投影事件]
    |
    | 餘額已重建
    v
[對帳事件]
    |
    | 差異已發現或已清除
    v
[營運 / 人工審核]
```

## API 到服務到持久化邊界

```text
[網頁客戶端]
    |
    | POST /api/orders
    v
[Spring Boot API]
    |
    | 驗證身分
    | 驗證欄位結構
    | 驗證交易對
    v
[訂單應用服務]
    |
    | 檢查冪等紀錄
    | 建立訂單意圖
    v
[領域服務]
    |
    | 風控檢查
    | 預留檢查
    v
[持久化邊界]
    |
    | 寫入訂單資料
    | 寫入預留資料
    | 寫入冪等紀錄
    v
[已提交的訂單狀態]
```

## 訂單事件流

```text
[已接受訂單]
    |
    | orderId
    | clientOrderId
    | 方向 / 價格 / 數量
    v
[訂單儲存層]
    |
    | 狀態 = 已接受
    | openQuantity = 原始數量
    v
[撮合閘道]
    |
    | 提交命令
    v
[C++ 撮合核心]
    |
    | 收進委託簿
    | 發出成交或撤單或拒絕事件
    v
[訂單狀態機]
    |
    | 未完成
    | 部分成交
    | 全部成交
    | 已撤單
    | 已拒絕
```

## 撮合事件流

```text
[Java 訂單服務]
    |
    | 寫入出站撮合命令
    v
[撮合閘道]
    |
    | 保持命令順序
    | 附加請求中繼資料
    v
[C++ 撮合核心]
    |
    | 擁有委託簿
    | 擁有價格時間優先規則
    | 擁有成交結果決定權
    v
[Java 事件收件匣]
    |
    | 持久化事件收件紀錄
    | 依 eventId / tradeId / sequence 去重
    v
[訂單 / 結算狀態機]
```

## 結算事件流

```text
[成交事件]
    |
    | orderId / tradeId / price / quantity
    v
[結算引擎]
    |
    | 計算基礎資產變動
    | 計算報價資產變動
    | 計算手續費
    | 計算預留扣用 / 釋放
    v
[結算儲存層]
    |
    | 持久化結算嘗試
    | 持久化結算鍵
    v
[帳本分錄]
    |
    | 寫入平衡分錄
    v
[訂單收尾器]
    |
    | 更新訂單成交狀態
    | 更新預留狀態
```

## 帳本事件流

```text
[結算引擎]
    |
    | 經濟結果
    v
[帳本分錄]
    |
    | 建立分錄主檔
    | 建立借方明細
    | 建立貸方明細
    v
[帳本提交]
    |
    | 分錄狀態 = 已提交
    v
[帳本消費方]
    |
    | 餘額投影
    | 對帳
    | 稽核報表
```

## 餘額投影流

```text
[已提交帳本分錄]
    |
    | 不可變的會計來源
    v
[餘額投影工作器]
    |
    | 重建可用餘額
    | 重建鎖定餘額
    | 重建總餘額
    v
[餘額投影儲存層]
    |
    | 持久化帳戶 / 資產餘額
    v
[資產查詢 API]
    |
    | 回傳可查詢餘額
    v
[對帳]
    |
    | 比對投影與帳本推導餘額
```

## 市場資料流

```text
[C++ 撮合核心]
    |
    | 權威的委託簿變化
    | 權威的成交事件
    v
[市場資料管線]
    |
    | 建立快照
    | 建立增量
    | 建立逐筆成交流
    | 建立行情摘要 / K 線
    v
[市場資料儲存層 / 快取]
    |
    | 持久化事件序號
    | 持久化快照狀態
    v
[REST 行情 API / WebSocket 推送]
    |
    | 發布公開市場資料
    v
[對帳]
    |
    | 驗證序號連續性
```

## 對帳流

```text
[對帳工作]
    |
    | 比對訂單與成交
    | 比對成交與結算
    | 比對結算與帳本
    | 比對帳本與餘額投影
    | 比對撮合事件與市場資料序號
    v
[差異報告]
    |
    | 分類差異
    | 建立人工審核項目
    v
[補償流程]
    |
    | 只能在明確批准後執行
    v
[稽核軌跡]
```

## 重複 / 重試 / 失敗處理流

```text
[重試中的客戶端 / 工作者]
    |
    | 相同的 clientOrderId / requestId / eventId
    v
[冪等儲存層]
    |
    | 回傳既有已接受訂單
    | 拒絕衝突內容
    v
[事件收件匣]
    |
    | 重複事件 -> 不執行副作用
    | 序號缺口 -> 暫停並告警
    | 暫時性失敗 -> 從檢查點重試
    v
[結算 / 帳本 / 投影]
    |
    | 副作用只執行一次
    v
[對帳]
    |
    | 驗證沒有靜默漂移
```

## 不能假裝完成的事

- 不要用畫面狀態、行情價格或人工推測假裝成交已發生。
- 不要在分錄尚未成功前直接改餘額，假裝結算已完成。
- 不要直接修改餘額投影來掩蓋帳本差異。
- 不要讓 Java 自己生成成交結果，假裝完成撮合。
- 不要從非正式資料源拼市場資料，假裝事件序號完整。
- 不要忽略重複事件而不留下去重依據。
- 不要在對帳失敗時靜默修復或直接清除異常。
