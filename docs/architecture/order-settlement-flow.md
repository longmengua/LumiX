# 訂單與結算流程

這份文件描述架構層級的現貨訂單與結算流程。目標是把一筆訂單從提交、預留、撮合、成交、結算到對帳的完整生命週期講清楚，並明確標出失敗與重試邊界。

這份文件本身必須可以單獨閱讀，不依賴其他文件補完主要流程。

## 共通規則

- `clientOrderId` 必須冪等。
- 送撮合前一定要先預留資金。
- `C++ 撮合核心` 對成交 / 拒絕 / 撤單事件有權威性。
- `結算引擎` 只能消費正式撮合事件，不能自己猜成交。
- `帳本分錄` 必須平衡。
- `對帳` 必須能回查訂單、預留、成交、分錄與投影。

## 買單正常流程

```text
[使用者]
    |
    | 提交 BUY BTC-USDT
    | 價格 = 60000
    | 數量 = 0.1
    | clientOrderId = buy-001
    v
[Spring Boot API]
    |
    | 驗證請求
    | 驗證身分
    | 驗證交易對
    v
[訂單服務]
    |
    | 檢查 clientOrderId 冪等
    | 建立訂單意圖
    v
[預留引擎]
    |
    | 鎖定 6000 USDT
    v
[訂單儲存層]
    |
    | 持久化訂單
    | 持久化預留
    | 訂單狀態 = 已接受
    v
[撮合閘道]
    |
    | 傳送命令到 C++ 撮合核心
    v
[C++ 撮合核心]
    |
    | 撮合訂單
    | 發出成交事件
    v
[結算引擎]
    |
    | 消費權威成交事件
    | 扣用已使用報價資產
    | 入帳基礎資產
    | 收取手續費
    | 釋放未使用預留
    v
[帳本分錄]
    |
    | 寫入平衡分錄
    v
[餘額投影]
    |
    | 更新可用餘額
    | 更新鎖定餘額
    | 更新總餘額
    v
[對帳]
```

## 部分成交

```text
[已接受訂單]
    |
    | 第一筆成交只覆蓋部分原始數量
    v
[結算引擎]
    |
    | 只結算已成交數量
    v
[帳本分錄]
    |
    | 寫入部分成交分錄
    v
[訂單儲存層]
    |
    | 訂單狀態 = 部分成交
    | 已成交數量增加
    | 剩餘數量保持未完成
    v
[預留引擎]
    |
    | 保留剩餘資金鎖定
    v
[對帳]
```

## 全部成交

```text
[未完成或部分成交訂單]
    |
    | 最後一筆成交吃掉剩餘數量
    v
[結算引擎]
    |
    | 結算最後一次成交
    v
[預留引擎]
    |
    | 扣用剩餘已使用預留
    | 若有零頭則釋放
    v
[訂單儲存層]
    |
    | 訂單狀態 = 全部成交
    | openQuantity = 0
    v
[帳本分錄]
    |
    | 最後分錄已提交
    v
[對帳]
```

## 成交前撤單

```text
[已接受訂單]
    |
    | 尚未發出任何成交事件
    v
[撤單請求]
    |
    | 傳送撤單命令
    v
[C++ 撮合核心]
    |
    | 移除掛單
    | 發出撤單事件
    v
[訂單儲存層]
    |
    | 訂單狀態 = 已撤單
    v
[預留引擎]
    |
    | 釋放全部鎖定資金
    v
[餘額投影]
    |
    | 鎖定餘額下降
    | 可用餘額上升
    v
[對帳]
```

## 部分成交後撤單

```text
[部分成交訂單]
    |
    | 部分數量已完成結算
    v
[撤單請求]
    |
    | 撤掉剩餘未完成數量
    v
[C++ 撮合核心]
    |
    | 對剩餘數量發出撤單事件
    v
[預留引擎]
    |
    | 釋放剩餘鎖定資金
    v
[訂單儲存層]
    |
    | 訂單狀態 = 已撤單
    | 已成交數量保持已記錄
    v
[對帳]
```

## 訂單被拒絕

```text
[使用者]
    |
    | 提交無效或被阻擋的訂單
    v
[Spring Boot API / 風控檢查]
    |
    | 驗證失敗或交易被阻擋
    v
[訂單儲存層]
    |
    | 訂單狀態 = 已拒絕
    | 沒有有效預留
    v
[撮合閘道]
    |
    | 不送出任何命令
    v
[對帳]
```

## 預留失敗

```text
[訂單服務]
    |
    | 請求預留
    v
[預留引擎]
    |
    | 餘額不足或儲存失敗
    v
[訂單儲存層]
    |
    | 訂單狀態 = 已拒絕 或 等待重試
    | 預留狀態 = 失敗
    v
[撮合閘道]
    |
    | 不送出任何撮合命令
    v
[對帳]
```

## 結算失敗

```text
[成交事件]
    |
    | 已收到權威成交事件
    v
[結算引擎]
    |
    | 建立結算嘗試
    v
[帳本分錄]
    |
    | 寫入失敗
    v
[待處理結算佇列]
    |
    | 用同一個結算鍵重試
    v
[帳本分錄]
    |
    | 只寫入一次平衡分錄
    v
[訂單儲存層]
    |
    | 只有在分錄提交後才標記結算完成
    v
[對帳]
```

## 重複成交事件

```text
[Java 事件收件匣]
    |
    | tradeId 已處理過
    v
[去重檢查]
    |
    | 偵測到重複成交
    v
[結算引擎]
    |
    | 跳過第二次副作用
    v
[訂單儲存層 / 帳本分錄]
    |
    | 只保留原始成交結果
    v
[對帳]
```

## 冪等處理

```text
[重試中的客戶端]
    |
    | 重送相同 clientOrderId
    v
[訂單服務]
    |
    | 載入冪等紀錄
    | 比對請求內容
    v
[冪等儲存層]
    |
    | 相同請求 -> 回傳既有訂單
    | 衝突請求 -> 拒絕
    v
[撮合閘道]
    |
    | 命令只提交一次
    v
[結算引擎]
    |
    | 也依 fillId / tradeId 去重
```

## 對帳

```text
[對帳工作]
    |
    | 比對訂單狀態與成交總量
    | 比對預留殘值與未完成數量
    | 比對結算結果與帳本分錄
    | 比對帳本分錄與餘額投影
    v
[異常報告]
    |
    | 人工審核
    | 補償只能在批准後執行
    v
[稽核軌跡]
```
