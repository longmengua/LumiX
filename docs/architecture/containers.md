## 容器責任

- 網頁應用程式：顯示交易、錢包、帳戶與管理畫面，不保存資金真相。
- REST 介面：負責驗證、授權、輸入檢查、冪等性與呼叫應用服務。
- 應用服務：負責交易邊界與流程協調。
- 交易核心：帳本、凍結、撮合、結算與風控的核心規則。
- Worker：負責 outbox 投遞、鏈上監聽、對帳與市場資料。
- PostgreSQL：資金與業務資料的真相來源。
- Redis：快取、短期鎖與限流，不是資金真相。
